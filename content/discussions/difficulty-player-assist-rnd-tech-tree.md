# 难度分层、玩家辅助与研发科技树随机性

> 这段讨论提出的问题是：游戏需要保留 LLM 研发的不确定性，但不能因为过硬核而让普通玩家大多数时候玩不通。

## 讨论摘要

玩家是否应该“开天眼”或获得更强提示，需要和难度模式一起讨论。简单难度可以让玩家拥有非常充裕的资金，使公司不太可能过早淘汰；也可以区分软核 / 硬核模式，让部分约束在低难度下被酌情放松。

可玩性上的倾向是：除非玩家操作特别差，最好多数局都能打通。讨论中用《瘟疫公司》作类比，意思不是复刻它的机制，而是强调策略游戏不能让玩家大部分时候被系统早早消灭，需要给玩家一定加强和容错。

研发系统仍然需要保留试错。优势科技树可以每局随机，玩家需要重新实验和推断，以表现 LLM 研发的抽卡感。这里的风险是，如果提示、天眼或系统辅助过强，游戏可能自动暴露最优解，变成单纯规划题。

科技树材料可以从现实 LLM 论文 idea 和已有口胡列表里整理，但不能只塞少量著名技术。现实材料进入游戏前需要筛掉不适合的内容，避免把不可靠、难辨认或还没确认的方向直接做成奖励。

模型规模和基础设施约束需要继续保留。目标参数规模不应任由玩家拍脑袋输入，而应有经验公式或科技树限制，否则会出现类似 20T dense 的不合理路线。

## 待确认问题

- 难度命名是否使用“简单 / 普通 / 硬核”，还是使用“软核 / 硬核”。
- “开天眼”具体指隐藏变量提示、推荐行动、失败回滚、SL 友好，还是更高的初始资源。
- 低难度下可以放松哪些约束：资金、竞争压力、训练失败惩罚、评估噪音、技术树随机性，还是基础设施硬限制。
- 现实 LLM 论文 idea 的来源池、筛选标准和维护方式尚未确定。
- 讨论中的“A 门”、梁圣论文和既有口胡列表需要提供具体材料后再整理。
